# []{#ch-system-autoconf}8.45. Autoconf-2.73 {#autoconf-2.73 .sect1}

::::::::::::::: {.wrap lang="en"}
::::::: {.package lang="en"}
The Autoconf package contains programs for producing shell scripts that can automatically configure source code.

:::::: segmentedlist
::::: seglistitem
::: seg
**Approximate build time:** [less than 0.1 SBU (about 0.4 SBU with tests)]{.segbody}
:::

::: seg
**Required disk space:** [25 MB]{.segbody}
:::
:::::
::::::
:::::::

::: {.installation lang="en"}
## 8.45.1. Installation of Autoconf {#installation-of-autoconf .sect2}

Prepare Autoconf for compilation:

``` userinput
./configure --prefix=/usr
```

Compile the package:

``` userinput
make
```

To test the results, issue:

``` userinput
make check
```

Install the package:

``` userinput
make install
```

### Short Descriptions

  ---------------------------------------------------- ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
  []{#autoconf}[[**autoconf**]{.command}]{.term}       Produces shell scripts that automatically configure software source code packages to adapt to many kinds of Unix-like systems; the configuration scripts it produces are independent---running them does not require the [**autoconf**]{.command} program
  []{#autoheader}[[**autoheader**]{.command}]{.term}   A tool for creating template files of C [*#define*]{.emphasis} statements for configure to use
  []{#autom4te}[[**autom4te**]{.command}]{.term}       A wrapper for the M4 macro processor
  []{#autoreconf}[[**autoreconf**]{.command}]{.term}   Automatically runs [**autoconf**]{.command}, [**autoheader**]{.command}, [**aclocal**]{.command}, [**automake**]{.command}, [**gettextize**]{.command}, and [**libtoolize**]{.command} in the correct order to save time when changes are made to [**autoconf**]{.command} and [**automake**]{.command} template files
  []{#autoscan}[[**autoscan**]{.command}]{.term}       Helps to create a `configure.in`{.filename} file for a software package; it examines the source files in a directory tree, searching them for common portability issues, and creates a `configure.scan`{.filename} file that serves as a preliminary `configure.in`{.filename} file for the package
  []{#autoupdate}[[**autoupdate**]{.command}]{.term}   Modifies a `configure.in`{.filename} file that still calls [**autoconf**]{.command} macros by their old names to use the current macro names
  []{#ifnames}[[**ifnames**]{.command}]{.term}         Helps when writing `configure.in`{.filename} files for a software package; it prints the identifiers that the package uses in C preprocessor conditionals \[If a package has already been set up to have some portability, this program can help determine what [**configure**]{.command} needs to check for. It can also fill in gaps in a `configure.in`{.filename} file generated by [**autoscan**]{.command}.\]
