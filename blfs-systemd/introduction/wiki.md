# Editor Notes {#editor-notes}

<div class="sect1" lang="en">
The BLFS Project has created a Wiki for editors to comment on pages and instructions at <a class="ulink" href="https://wiki.linuxfromscratch.org/blfs/wiki">https://wiki.linuxfromscratch.org/blfs/wiki</a>.

When <span class="emphasis"><em>editor notes</em></span> are present, a link appears in the form <code class="literal">https://wiki.linuxfromscratch.org/blfs/wiki/pkgname</code> right below the dependency list. The idea behind the editor notes is to give additional information about the package and/or its build instructions, common pitfalls or maybe even more sophisticated configuration for special cases of use.

The vast majority of the packages do not have editor notes.

<div class="admon note">
![\[Note\]](../images/note.png)

### Note

The <span class="emphasis"><em>editor notes</em></span> might be outdated. Even though the pages should be reviewed when a package is updated, it might happen that there are notes referring to an obsolete version and therefore, the notes might be out of date. Always check the date of the notes and more importantly, the version of the package the notes refer to.
