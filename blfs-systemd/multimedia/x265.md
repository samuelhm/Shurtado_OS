## Installation of x265 {#installation-of-x265}

First, fix building this package for 32-bit platforms:

```bash
sed -i 's/FORMAT_ELF/UNIX64 \&\& FORMAT_ELF/' source/common/x86/cpu-a.asm
```

Install <span class="application">x265</span> by running the following commands:

```bash
mkdir bld &&
cd    bld &&

cmake -D CMAKE_INSTALL_PREFIX=/usr        \
      -D GIT_ARCHETYPE=1                  \
      -W no-dev                           \
      ../source                           &&
make
```

This package does not come with a test suite.

To install the package, first remove any old library versions. After installation, remove a static library. As the <code class="systemitem">root</code> user:

```bash
make install &&
rm -vf /usr/lib/libx265.a
```
</div>

<div class="commands" lang="en">
## Command Explanations {#command-explanations}

*`-D GIT_ARCHETYPE=1`*: This switch is needed because the build system will not generate a pkg-config file or install a shared library unless <a class="xref" href="../general/git.md" title="Git-2.54.0">git-2.54.0</a> is installed. Git isn't required for anything else in this package, so this switch is used to work around it for users who do not want to install Git as a build dependency.

*`-W no-dev`*: This switch is used to suppress warnings intended for the package developers.

<span class="command"><strong>rm -vf /usr/lib/libx265.a</strong></span>: BLFS does not recommend using static libraries.
</div>
